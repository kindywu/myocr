# 环境与运行说明

本文档说明 MyOCR 的运行环境、Android 构建方式、PC 接收服务启动步骤，以及真机联调流程。

## 环境要求

### Android 端

- Android Studio 可正常打开 Gradle 工程
- Android SDK 版本需满足 `compileSdk 37`
- 目标设备系统版本需满足 `minSdk 35`
- 真机需要可用相机
- 若使用朗读功能，系统需提供可用的 TTS 引擎与中文语音数据

### PC 端

- Go 运行环境
- 手机与 PC 在同一局域网
- 本机防火墙允许指定端口访问

## Android 构建与运行

### Android Studio

1. 打开仓库根目录
2. 等待 Gradle 同步完成
3. 连接真机并选择 `app` 配置
4. 运行应用

### 命令行构建

在仓库根目录执行：

```bash
./gradlew assembleDebug
```

如需安装到已连接设备，可执行：

```bash
./gradlew installDebug
```

## PC 接收服务启动

进入服务目录后执行：

```bash
cd server
go run main.go
```

默认端口为 `8080`。如果需要自定义端口：

```bash
go run main.go 8081
```

启动成功后，控制台会输出：

- 当前监听端口
- 结果输出目录
- 本机可访问的局域网 IP
- 手机端建议填写的地址

## 联调步骤

1. 在 PC 上启动 Go 接收服务
2. 记录控制台输出的局域网地址，例如 `http://192.168.1.100:8080`
3. 在手机上启动 MyOCR
4. 首次进入授予相机权限
5. 在服务器输入框中填写 `192.168.1.100:8080` 或完整地址 `http://192.168.1.100:8080`
6. 拍照识别或从相册选择图片
7. 识别完成后点击“上传”
8. 在 PC 的 `server/ocr_results/` 目录中查看结果文件

## 输出位置

Go 服务会在当前工作目录下创建 `ocr_results/` 目录，并写入：

- 单次结果文件，例如 `ocr_20260714_101530.txt`
- 聚合日志文件 `_all_results.log`

建议在 `server/` 目录中启动服务，以便输出统一保存在 `server/ocr_results/`。

## 朗读功能准备

应用内朗读依赖系统 TTS：

- 默认尝试使用 `TextToSpeech` 标准接口
- 若标准接口不可用，会退回到系统 `SPEAK` Intent
- 如仍无法朗读，应用会引导用户打开 TTS 设置或安装语音数据

建议在联调前先确认：

- 已选择默认 TTS 引擎
- 已安装中文语音数据
- 系统设置页中能正常试听中文语音

## 常见问题

### Android Studio 同步失败

- 检查本地 Gradle 与 Android SDK 是否完整
- 确认网络能下载 Android 依赖

### 手机无法上传到 PC

- 确认手机和 PC 在同一网段
- 确认输入的是 PC 的局域网地址而不是 `localhost`
- 确认 PC 防火墙未拦截端口
- 确认 Go 服务已经启动

### OCR 无结果

- 尽量使用清晰、光线充足的图片
- 检查目标图片中是否确实包含可识别文字
- 若从相册选择图片，确认图片文件可正常读取

### 朗读不可用

- 打开系统 TTS 设置，检查默认引擎
- 安装或更新中文语音数据
- 某些 ROM 对标准 TTS 支持不完整，可尝试系统级语音设置后重试

### 上传成功但看不到文件

- 确认服务是在 `server/` 目录中启动
- 检查 `ocr_results/` 目录是否创建成功
- 查看控制台日志中记录的具体文件名
